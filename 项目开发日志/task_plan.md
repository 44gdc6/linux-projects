## 关键问题
1. `F407` 当前使用的 `UART5(PC12/PD2)` 是否已经接出并方便观察串口日志。
2. 本轮先用 `can-utils` 做主站联调，暂不编写 Linux 用户态守护程序。

## 已做决策
| 决策 | 理由 |
|------|------|
| 采用统一节点协议模板，主站为 `i.MX6ULL` | 方便总线扩展到 2 个以上节点，且最符合“多节点挂载”目标 |
| 使用标准帧 `11-bit ID` | 当前三端调试成本最低，`candump/cansend` 也最好用 |
| 默认波特率固定为 `500 kbps` | F103 现有链路已验证成功，F407 直接对齐即可 |
| 节点 ID 先定为 `F103=0x01`、`F407=0x02` | 先完成最小闭环，后续若要扩节点再扩展编号 |
| 先保留每个工程内部的 `CAN_Link_*` 命名 | 尽量少改现有工程结构，降低移植风险 |
